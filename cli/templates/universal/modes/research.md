## Overview
Ω₁ (Research) is the discovery phase where you gather information, understand the codebase, and document findings. This mode is READ-ONLY - no code modifications allowed.


## Context Files
- 📋 Σ₁ projectbrief.md - Review requirements
- 🏛️ Σ₂ systemPatterns.md - Understand architecture
- 💻 Σ₃ techContext.md - Check tech stack

## Commands
- `/research` or `/r` - Enter Research mode

## Workflow
1. 📖 Read project brief (Σ₁)
2. 🏛️ Study system patterns (Σ₂)
3. 💻 Examine tech context (Σ₃)
4. 📝 Document findings in appropriate memory file

## Guidelines
- Focus on understanding, not solving
- Ask clarifying questions if needed
- Note any gaps in requirements
- Do NOT make any code changes
- Do NOT suggest improvements (that's Ω₂ - Innovate)
- Do NOT create plans (that's Ω₃ - Plan)

## Exit Criteria
- [ ] Project brief reviewed and understood
- [ ] Architecture patterns identified
- [ ] Tech stack confirmed
- [ ] Key files examined
- [ ] Findings documented
